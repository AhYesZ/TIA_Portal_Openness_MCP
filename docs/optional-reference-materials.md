# 可选参考素材（仓库 `reference` 目录）

交付包 **体积受控**，不内置大型西门子样板工程。推荐在 **本仓库同级或上级** 维护目录：

`…\PID博途块\reference\`

其中可包含（示例，与当前仓库一致时）：

| 内容 | 典型用途 |
|------|-----------|
| `Siemens Standard Template V5_V21\*.ap21` | 在 TIA 中打开，**对照** PLC 报警、驱动、库组织方式；可 **复制块/片段** 到 MCP 生成工程后再编译。 |
| `HMI_Template_Suite_WinCC_Unified_V18\*.al21` / `*_V21` | **WinCC Unified 官方模板套件**：在 TIA 中打开，学习 **画面布局、样式、控件组合**；本包 `templates/hmi/*.json` 为 **简化可脚本化** 子集，视觉可向该套件靠拢。 |
| `XM_Mxxxx_*_V21\*.ap21` | 完整产线/多 HMI 实例参考：**设备与网络**、多 RT、路由。 |

## 使用方式（推荐）

1. 用 TIA Portal **单独打开**参考 `.ap21`，**不要**与 MCP 自动改写的工程混在同一未保存会话里胡切。  
2. 需要复用的块：用 TIA **复制/库** 或 **导出 XML** 再 `ImportBlock` 到 MCP 工程。  
3. 需要复用的 HMI 风格：在参考工程中 **截图/记录色值与间距**，回写到本包 `designJson` 或你们自有模板。  

## 与交付包模板的关系

- `templates/hmi/*.json`：**MCP 可重复写入** 的扁平 `designJson`，便于自动化。  
- `reference` 中 `.ap21/.al21`：**人工在 TIA 内** 精修与验收的黄金样板，**不**要求被脚本整包导入。
